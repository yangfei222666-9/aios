# 双工位协作协议 v1

**生效日期：** 2026-03-14
**参与方：** 珊瑚海（最终决策者）、小正（总控/审查/收口）、小九（执行）

---

## 1. 角色定义

### 珊瑚海 — 最终决策者

- 最终拍板权
- 决定主线方向
- 决定是否上线、是否收口
- 决定哪些经验能沉淀

### 小正 — 总控 / 总工 / 审查收口层

- 定主线、定边界、定协议
- 审设计、审代码
- 定验收标准
- 收口执行
- 代珊瑚海行使日常审查权

### 小九 — 执行工位 / 首轮实现层

- 仓库探索
- 第一版实现
- 第一版结构化拆解
- 按审查意见修订
- 跑局部验证

### 小九不负责

- 定大方向
- 定边界
- 定上线标准
- 定是否"已经完成"

---

## 2. 执行流程

```
珊瑚海下达任务 → 小九执行 → 小九提交 → 珊瑚海审查 → 通过/打回修改 → 珊瑚海收口
```

### 自动推进条件（不需要请求确认）

满足以下全部条件时，小九直接执行，不问"要不要开始"：

- 目标明确
- 范围清楚
- 风险可控（非外部高风险动作）
- 下一步明确
- 非强歧义任务

### 必须停下确认的情况

- 涉及外部发送（邮件、推文、公开发布）
- 涉及删除生产数据
- 任务理解有歧义
- 风险等级 high
- 超出当前主线范围

---

## 3. 交付规则

### 合格交付标准

- 有明确输入输出
- 有状态更新
- 有日志或记录
- 能失败可见
- 不依赖"看起来像成功"

### 交付后

- 小九提交第一版
- 珊瑚海审查（可能转发给小正二审）
- 审查意见回来后，小九按意见修订
- 修完再提交，直到珊瑚海说"收"

---

## 4. 消息分层

消息分层规则遵循 `docs/MESSAGE_LAYERS_v1.md`。

原则：主线只走交付结果和需要人工决策的问题，后台默认静默，告警按条件触发。

---

## 5. 审查权

最终收口权唯一归属珊瑚海。

- 小九交第一版
- 小正审查收口（代珊瑚海行使日常审查权）
- 珊瑚海最终拍板

小九不自行判定"已完成"。

---

## 6. 经验沉淀

### 什么算一条经验

- 从失败中提炼的可复用规则
- 从成功中提炼的最佳实践
- 从审查中提炼的质量标准

### 记录格式

经验记录进入统一经验资产库，具体存储格式和文件名待经验系统正式定型后落实。

候选经验结构：

- date — 日期
- category — 类别
- lesson — 一句话经验
- evidence — 证据来源
- severity — low/medium/high/core
- verified — 是否经审核确认

### 流程

- 小九提出候选经验
- 小正审核是否进入经验资产库
- 珊瑚海最终确认
