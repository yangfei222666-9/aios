## 1. 选题入口

每次只从这 4 类里选，不要发散：

### 进展型
今天做了什么，解决了什么问题

**例子：**
- 今天把 Skill 自动创建的 MVP 规格书写完了
- 修复了任务队列的回写失败问题
- Agent 协作从串行改成并行，效率提升 3 倍

### 方法型
某个坑怎么排查，某个系统怎么搭

**例子：**
- 如何排查 Agent 执行断点
- 如何搭建本地 RAG 记忆系统
- 如何用易经状态引擎做系统决策

### 认知型
你对 AIOS / Agent / Skill / 自动化的新理解

**例子：**
- 我终于明白，做 AIOS 最怕的不是不会写代码
- 为什么很多人做自动化，最后都死在"看起来很强"
- 先修主链路，再扩功能，能少走很多弯路

### 成长型
从混乱到收敛、从报错到闭环的过程

**例子：**
- 我把 AI 项目从混乱拉回来了，只做了一件事
- 从 37 个 Agent 混乱运行到有序协作的过程
- 如何从"假运行"走向"真闭环"

**规则：**
- ✅ 一次只做一个主题
- ✅ 一条内容只讲一个核心点
- ✅ 能讲"变化前/变化后"就不要只讲概念

## 2. 内容生产流程

固定成 6 步：

### 第 1 步：先定一句话主题

**格式：** 这条内容我只想让别人记住什么？

**例子：**
- 我做 AIOS，不是为了炫技，是为了把个人工作流变成可持续系统
- 先修主链路，再扩功能，能少走很多弯路
- 失败任务 80% 自动修复，不是靠运气，是靠闭环

### 第 2 步：写 3 个钩子标题

**例子：**
- 我终于明白，做 AIOS 最怕的不是不会写代码
- 为什么很多人做自动化，最后都死在"看起来很强"
- 我把 AI 项目从混乱拉回来了，只做了一件事

**技巧：**
- 用"我终于明白"、"为什么"、"只做了一件事"等钩子
- 制造冲突感、好奇感
- 避免空泛概念，用具体场景

### 第 3 步：写正文骨架

**固定结构：**
1. 我遇到了什么问题
2. 我以前是怎么想错的
3. 我现在怎么做
4. 结果有什么变化
5. 给别人什么启发

**例子：**

```
【问题】
我做 AIOS 两周，37 个 Agent 跑起来了，但经常出现"看起来成功，实际没做"的假运行。

【想错】
我以为只要 Agent 返回"完成"就算成功，结果发现很多任务根本没真正执行。

【现在】
我加了三层验证：
1. 执行前检查输入
2. 执行中记录日志
3. 执行后验证输出

【变化】
假运行从 30% 降到 5%，任务可靠性提升 6 倍。

【启发】
做 AIOS 不是"能跑"就行，要"能验证"。
```

## 3. 平台适配规则

### 小红书

**重点：**
- 像"真实项目复盘"
- 讲过程、讲踩坑、讲变化
- 少空话，多细节

**适合：**
- 项目进展
- 排错记录
- AIOS 搭建心得
- 个人成长感悟

**结构：**
1. 标题先抛冲突
2. 正文前半讲问题
3. 中间讲你怎么修
4. 结尾给结论或方法

**例子：**
```
🔥 我做 AIOS 两周，踩了个大坑

【问题】37 个 Agent 跑起来了，但很多任务"看起来成功，实际没做"

【以前】我以为 Agent 返回"完成"就算成功

【现在】加了三层验证：执行前检查、执行中记录、执行后验证

【结果】假运行从 30% 降到 5%

💡 做 AIOS 不是"能跑"就行，要"能验证"
```

### 抖音

**重点：**
- 前 3 秒抓人
- 句子短
- 口语化
- 节奏快

**适合：**
- 一句话观点
- 快速展示成果
- bug 修复前后对比
- 个人 AI 系统演示

**结构：**
1. 开头先打痛点
2. 中间给画面/证据
3. 结尾给一句记忆点

**例子：**
```
做 AIOS 两周，我踩了个大坑。

37 个 Agent 跑起来了，但很多任务"看起来成功，实际没做"。

后来我加了三层验证，假运行从 30% 降到 5%。

```

**重点：**
- 观点要浓
- 句子要短
- 适合连续发帖

**适合：**
- 结论

- 产品思考
- AI Agent 观察

**结构：**
1. 先一句核心观点
2. 再补 2-4 条理由
3. 最后收束一句

## 4. 发布检查清单

发之前只检查这几个：

### 内容质量
- [ ] 这条内容是不是只有一个核心点
- [ ] 有没有真实细节，不是空泛感想
- [ ] 有没有"前后变化"或"结果"

### 标题和开头
- [ ] 标题够不够直接
- [ ] 开头能不能让人停下来

### 平台适配
- [ ] 小红书版本有没有共鸣感
- [ ] 抖音版本有没有口语感
- [ ] 标签有没有乱堆

### 风险检查
- [ ] 有没有敏感表述
- [ ] 有没有夸大表述
- [ ] 有没有平台容易误判的话

## 5. 数据反馈与迭代

每条内容发布后，只看 4 个问题：

### 有没有点击
**说明：** 标题/封面行不行

**改进：**
- 标题不够直接 → 改成更具体的场景
- 封面不够吸引 → 加数字、对比、冲突

### 有没有看完
**说明：** 开头和结构行不行

**改进：**
- 开头不够抓人 → 先抛痛点或冲突
- 结构太散 → 用固定骨架（问题→解决→结果）

### 有没有互动
**说明：** 观点有没有打到人

**改进：**
- 内容太空 → 加真实细节和数据
- 观点不够浓 → 提炼成一句话记忆点

### 有没有私信/收藏
**说明：** 内容有没有实际价值

**改进：**
- 没有方法论 → 补充具体步骤
- 没有可复用性 → 加模板或清单

### 复盘原则

不要问"为什么我没火"，只问：

- 是标题问题？
- 是选题问题？
- 是内容太空？
- 是平台不适配？
- 是证据不够？
- 是表达不够短？

### 迭代原则

- 数据差，不一定是你不行，先改标题和开头
- 有播放没互动，改内容密度
- 有互动没转化，改结尾引导
- 某类内容连续有效，就做成系列

## 使用说明

### 第一次使用
1. 复制"每日执行模板"到 `memory/content-production/YYYY-MM-DD.md`
2. 按模板填写
3. 发布后记录数据
4. 24h 后复盘

### 日常使用
1. 每天早上或晚上，选一个主题
2. 用 6 步流程生产内容
3. 用检查清单检查
4. 发布
5. 记录数据
6. 定期复盘

### 复盘频率
- 每条内容发布后 24h 复盘一次
- 每周复盘一次（看哪类内容有效）
- 每月复盘一次（调整选题方向）

## 附录：常见问题

### Q: 我不知道写什么怎么办？
A: 从"今天做了什么"开始，用进展型选题。

### Q: 标题不知道怎么起？
A: 用这 3 个模板：
- 我终于明白，XXX
- 为什么很多人 XXX，最后都 XXX
- 我把 XXX 从 XXX 拉回来了，只做了一件事

### Q: 内容太长怎么办？
A: 只讲一个核心点，其他的拆成下一条。

### Q: 数据不好怎么办？
A: 先改标题和开头，再改内容密度，最后改选题。

### Q: 要不要每天都发？
A: 不用。先保证质量，再考虑频率。一周 2-3 条高质量内容比每天 1 条低质量内容更有效。
