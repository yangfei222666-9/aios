# 太极OS Dashboard - 前端需求文档

**版本：** v1.0  
**最后更新：** 2026-03-10

---

## 项目概述

太极OS Dashboard 是一个可视化监控和管理界面，用于展示系统状态、Agent 运行情况、任务执行记录和系统健康度。

**目标用户：** 系统管理员、开发者

**核心价值：** 让用户能看见系统在做什么、运行得如何、有什么问题

---

## 页面结构

### 1. 总览页（Dashboard）
**路由：** `/`

**功能：**
- 显示系统整体健康度（0-100 分）
- 显示当前运行的 Agent 数量
- 显示任务队列状态（待处理/执行中/已完成/失败）
- 显示最近 24 小时的关键指标趋势图
- 显示最近的告警（最多 5 条）

**关键指标卡片：**
- Evolution Score（进化分数）
- Active Agents（活跃 Agent 数）
- Task Success Rate（任务成功率）
- System Uptime（系统运行时间）

### 2. Agent 管理页
**路由：** `/agents`

**功能：**
- 列出所有 Agent（名称、类型、状态、最后运行时间）
- 支持搜索和筛选（按状态、类型）
- 点击 Agent 查看详情
- 显示 Agent 统计（总任务数、成功率、平均执行时间）

**Agent 状态：**
- `active` - 活跃
- `idle` - 空闲
- `failed` - 失败
- `disabled` - 禁用

**Agent 类型：**
- `learning` - 学习型
- `executor` - 执行型
- `monitor` - 监控型

### 3. Agent 详情页
**路由：** `/agents/:agentId`

**功能：**
- 显示 Agent 基本信息（名称、类型、描述、创建时间）
- 显示 Agent 统计数据
- 显示最近执行的任务列表（最多 20 条）
- 显示执行时间趋势图（最近 7 天）
- 显示成功率趋势图（最近 7 天）

### 4. 任务队列页
**路由：** `/tasks`

**功能：**
- 列出所有任务（ID、类型、状态、优先级、创建时间）
- 支持搜索和筛选（按状态、类型、优先级）
- 点击任务查看详情
- 显示队列统计（待处理/执行中/已完成/失败）

**任务状态：**
- `pending` - 待处理
- `running` - 执行中
- `completed` - 已完成
- `failed` - 失败

**任务类型：**
- `code` - 代码相关
- `analysis` - 分析相关
- `monitor` - 监控相关
- `learning` - 学习相关

**优先级：**
- `high` - 高
- `normal` - 普通
- `low` - 低

### 5. 任务详情页
**路由：** `/tasks/:taskId`

**功能：**
- 显示任务基本信息（ID、类型、状态、优先级、创建时间、完成时间）
- 显示任务描述
- 显示执行结果（成功/失败信息）
- 显示执行日志（如有）
- 显示关联的 Agent

### 6. Skill 管理页
**路由：** `/skills`

**功能：**
- 列出所有 Skill（名称、版本、状态、调用次数）
- 支持搜索和筛选（按状态）
- 点击 Skill 查看详情
- 显示 Skill 统计（总数、活跃数、草稿数）

**Skill 状态：**
- `production` - 生产环境
- `draft` - 草稿
- `testing` - 测试中
- `deprecated` - 已废弃

### 7. Skill 详情页
**路由：** `/skills/:skillId`

**功能：**
- 显示 Skill 基本信息（名称、版本、描述、创建时间）
- 显示使用统计（调用次数、成功率、最近使用时间）
- 显示触发条件
- 显示使用历史（最近 20 次调用）

### 8. 系统日志页
**路由：** `/logs`

**功能：**
- 显示系统日志（时间、级别、来源、消息）
- 支持搜索和筛选（按级别、来源、时间范围）
- 支持实时刷新
- 支持导出日志

**日志级别：**
- `DEBUG` - 调试
- `INFO` - 信息
- `WARN` - 警告
- `ERROR` - 错误
- `CRITICAL` - 严重

### 9. 告警管理页
**路由：** `/alerts`

**功能：**
- 显示所有告警（时间、级别、来源、消息、状态）
- 支持搜索和筛选（按级别、状态、时间范围）
- 支持标记为已读/已处理
- 显示告警统计（总数、未处理数、严重告警数）

**告警级别：**
- `INFO` - 信息
- `WARN` - 警告
- `CRIT` - 严重

**告警状态：**
- `new` - 新告警
- `acknowledged` - 已确认
- `resolved` - 已解决

### 10. 设置页
**路由：** `/settings`

**功能：**
- 系统配置（刷新间隔、日志保留时间）
- 通知设置（告警通知方式）
- 主题设置（亮色/暗色）

## 通用 UI 组件

### 1. 导航栏（Navbar）
- Logo + 项目名称
- 主导航菜单（总览、Agent、任务、Skill、日志、告警、设置）
- 用户信息（可选）
- 系统状态指示器（健康/警告/错误）

### 2. 侧边栏（Sidebar）
- 折叠/展开功能
- 导航菜单
- 快速统计卡片

### 3. 状态徽章（Badge）
- 不同状态用不同颜色
- 支持图标 + 文字

### 4. 数据表格（Table）
- 支持排序
- 支持分页
- 支持搜索
- 支持筛选
- 支持导出

### 5. 图表（Charts）
- 折线图（趋势）
- 柱状图（对比）
- 饼图（占比）
- 仪表盘（健康度）

### 6. 卡片（Card）
- 标题 + 内容
- 支持操作按钮
- 支持加载状态

### 7. 加载状态（Loading）
- 骨架屏
- 加载动画
- 错误提示

## 交互流程

### 1. 查看系统状态
```
用户访问首页 → 显示总览页 → 查看关键指标 → 点击指标卡片 → 跳转到详情页
```

### 2. 查看 Agent 详情
```
用户访问 Agent 页 → 浏览 Agent 列表 → 点击 Agent → 查看详情 → 查看任务历史
```

### 3. 查看任务详情
```
用户访问任务页 → 浏览任务列表 → 筛选任务 → 点击任务 → 查看详情 → 查看执行日志
```

### 4. 处理告警
```
用户访问告警页 → 查看未处理告警 → 点击告警 → 查看详情 → 标记为已处理
```

## 响应式设计

### 桌面端（>= 1024px）
- 显示完整侧边栏
- 多列布局
- 完整表格

### 平板端（768px - 1023px）
- 可折叠侧边栏
- 两列布局
- 简化表格

### 移动端（< 768px）
- 隐藏侧边栏，使用汉堡菜单
- 单列布局
- 卡片式列表

---

## 性能要求

- 首屏加载时间 < 2 秒
- 页面切换 < 500ms
- 数据刷新 < 1 秒
- 支持实时更新（WebSocket 或轮询）

---

## 浏览器兼容性

- Chrome >= 90
- Firefox >= 88
- Safari >= 14
- Edge >= 90

## 技术建议

### 推荐技术栈
- **框架：** React 18+ / Vue 3+ / Next.js
- **UI 库：** Ant Design / Material-UI / Tailwind CSS
- **图表库：** ECharts / Chart.js / Recharts
- **状态管理：** Redux / Zustand / Pinia
- **HTTP 客户端：** Axios / Fetch API
- **实时通信：** WebSocket / Socket.io
