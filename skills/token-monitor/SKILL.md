# Token Monitor Skill

**实时监控 Token 使用量，自动优化策略，避免爆炸。**

## 触发词

- "token 使用量"
- "token 监控"
- "token 报告"
- "token 成本"

## 核心功能

### 1. 实时监控
- 每次请求后自动记录 token 使用量
- 累计统计（今日/本周/本月）
- 成本估算（基于模型定价）

### 2. 智能告警
- 接近限额时提前警告
- 异常使用模式检测
- 成本超标提醒

### 3. 自动优化
- 建议切换更便宜的模型
- 识别可压缩的 prompt
- 推荐批量处理策略

## 使用方式

### 查看当前使用量
```
token 使用量
```

### 查看详细报告
```
token 报告
```

### 设置告警阈值
```
设置 token 告警阈值为 100k
```

```

💡 优化建议：

```

## 监控指标

### 基础指标
- 总 token 数
- 输入 token 数
- 输出 token 数
- 请求次数

### 成本指标
- 每次请求成本
- 每日成本
- 每周成本
- 每月成本

### 效率指标
- 平均每次请求 token 数
- Token 使用趋势
- 成本效率比

## 告警规则

### 黄色告警（提醒）
- 日使用量超过平均值 50%
- 单次请求超过 10k tokens
- 成本超过预算 70%

### 红色告警（警告）
- 日使用量超过平均值 100%
- 单次请求超过 50k tokens
- 成本超过预算 90%

### 紧急告警（阻断）
- 接近 API 限额
- 成本超出预算
- 异常使用模式

## 优化策略

### 1. 模型选择
```
简单任务 → Claude Haiku ($0.25/1M tokens)
中等任务 → Claude Sonnet ($3/1M tokens)
复杂任务 → Claude Opus ($15/1M tokens)
```

### 2. Prompt 优化
- 移除冗余信息
- 使用更简洁的表达
- 启用 prompt 缓存

### 3. 批量处理
- 合并多个小请求
- 使用流式输出
- 启用响应缓存
