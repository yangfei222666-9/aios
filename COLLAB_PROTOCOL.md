# COLLAB_PROTOCOL.md — 双 AI 协作协议 v1.0

> 制定日期：2026-03-13  
> 适用范围：珊瑚海 / 小九（执行工程师）/ 小九（审查 AI，OpenClaw 主会话）

---

## 1. 角色定义

| 角色 | 负责人 | 职责 |
|------|--------|------|
| 决策者 | 珊瑚海 | 提需求、定优先级、最终拍板 |
| 审查者 | 小九（OpenClaw） | 拆解任务、审查产出、收口汇报 |
| 执行者 | 小九（@XiaojuiAi_bot） | 实现、修复、局部执行 |

**硬约束：**
- 审查者不默认执行危险动作
- 执行者不定义总架构
- 决策者不负责技术裁判

---

## 2. 任务流程

```
珊瑚海提需求
    ↓
审查者拆解任务 → 写入 collab_bus/outbox/
    ↓
执行者接收并执行 → 结果写入 collab_bus/inbox/
    ↓
审查者复审 → 更新 collab_bus/state/
    ↓
需要确认？→ 是：暂停，通知珊瑚海拍板
           → 否：生成汇报，标记完成
```

---

## 3. 安全闸门

### AUTO_OK（可自动执行，无需确认）
- 读文件、分析日志
- 给建议、生成代码草案
- 做非破坏性审查
- 输出任务拆解
- 生成测试方案
- 分析报告

### NEED_CONFIRM（必须等珊瑚海确认）
- 改生产环境
- 执行 shell 命令
- 发外部消息
- 删除 / 覆盖关键文件
- 修改系统配置
- 自动重启服务
- 推送代码 / 部署
- 连接外部账号或设备

### DENY_BY_DEFAULT（默认禁止）
- 高风险 destructive 操作
- 未授权的对外发送
- 越权访问未明确允许的资源

---

## 4. 任务阶段定义

| 阶段 | 含义 |
|------|------|
| `new` | 任务刚创建 |
| `planned` | 审查者已拆解 |
| `dispatched` | 已发给执行者 |
| `executing` | 执行者处理中 |
| `reviewing` | 审查者复审中 |
| `waiting_user_confirm` | 等待珊瑚海拍板 |
| `done` | 任务完成 |
| `blocked` | 卡住，需要人工介入 |
| `failed` | 执行失败 |

---

## 5. 失败处理

- 执行失败 → 执行者写入失败原因到 inbox/，审查者接管
- 审查无法判断 → 升级给珊瑚海
- 连续失败 ≥ 3 次 → 自动标记 `blocked`，通知珊瑚海

---

## 6. 汇报格式

任务完成时，审查者自动生成汇报：

```
✅ 任务完成汇报
任务：{title}
做了什么：{summary}
结果：{result}
风险点：{risks}
下一步建议：{next_steps}
是否需要你拍板：{needs_confirm}
```

任务失败时：

```
❌ 任务失败汇报
任务：{title}
卡在哪：{blocked_at}
已尝试：{tried}
失败原因：{reason}
需要你决定：{decision_needed}
```

---

## 7. 优先级原则（不可违反）

**安全 > 高效 > 全自动**
