### 3. health_monitor 监控规则需改进

**发现时间：** 2026-03-12 16:44

**问题描述：**
- health_monitor 将 `tasks_total = 0` 的 Agent 判定为"成功率低于 50%"
- 导致大量"从未执行"的 Agent 被误报为"低成功率故障"
- 无法区分"跑了很多但全失败"和"从未运行"

**影响范围：**
- ❌ 健康度评分不准确
- ❌ 告警噪音过大

**定性：**
- P1 破口（影响可信度）
- 监控规则缺陷

**根本原因：**
- 监控规则未区分 `never_run` 和 `unhealthy` 状态
- 将 `tasks_total = 0` 按失败处理

**观察期后处理方案：**
1. 将 Agent 状态分成 3 类：
   - **healthy** - `tasks_total > 0` 且成功率达标
   - **unhealthy** - `tasks_total > 0` 且成功率低于阈值
   - **no_data / never_run** - `tasks_total = 0`
2. 改进告警文案：
   - ❌ "成功率低于 50% (0.0%)"
   - ✅ "未执行过，暂无样本"
   - ✅ "状态：no_data / never_run"

**一句话总结：**
> health_monitor 需要区分"从未运行"和"低成功率"，避免误报。

**一句话总结：**
