## 7. GitHub 学习收获

### hermes-agent Skill 系统研究
- **核心发现：** Agent-Curated Memory（Agent 主动创建 Skill）
- **5 个核心问题的答案：**
  1. 怎么发现可抽象的 Skill？- Agent 主动判断（5+ 工具调用、错误克服、用户纠正、非平凡工作流）
  2. 怎么生成 Skill 草案？- Agent 直接调用 `skill_manage(action='create', content='...')`
  3. 怎么验证 Skill？- 三层验证（格式验证 + 安全扫描 + 回滚机制）
  4. 怎么注册/索引？- 直接写入文件系统，启动时扫描构建内存索引，渐进式加载（Level 0/1/2）
  5. 怎么根据使用结果继续改进？- Agent 主动调用 `skill_manage(action='patch', ...)` 或 `edit`

- **对太极OS 的建议：**
  1. 融合两种模式 - 自动检测（太极OS）+ Agent 主动创建（hermes-agent）
  2. 采用渐进式加载 - Level 0/1/2
  3. 集成安全扫描 - 参考 tools/skills_guard.py
  4. Patch 优先策略 - feedback loop 优先生成 patch
  5. 平台兼容性字段 - 在 SKILL.md frontmatter 中加入 platforms

### Parlant 动态上下文匹配引擎
- **核心洞察：** 不把所有规则塞进一个大 prompt，而是在每个对话轮次动态匹配相关的规则、工具、知识
- **关键原则：** "Adding more rules makes the agent smarter, not more confused — because the engine filters context relevance, not the LLM."
- **对太极OS 的建议：** 在 Skill 自动创建 MVP 中引入 skill_trigger_spec（激活信号、排除条件、优先级、必需的上下文字段）
