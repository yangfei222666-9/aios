# 太极OS 进度追踪

**核心目标：** 把太极OS 做成可靠的个人 AI Agent 系统 — 能稳定跑任务、看见状态、处理异常、积累经验。

**最后更新：** 2026-03-13

---

## P0 — 真链跑通（当前最高优先级）

真链 = 端到端可验证、可复现、有真实数据流过的链路。

| 链路 | 状态 | 说明 |
|------|------|------|
| 心跳 → 健康检测 → 告警 → Telegram 推送 | ✅ 已验证 | 唯一完整闭环 |
| Task Queue 入队/查询 | ✅ 已验证 | enqueue_task / list_tasks_by_status |
| Memory Server 启动/查询 | ✅ 已验证 | 端口 7788，embedding 热加载 |
| 状态词表统一 | ✅ 已验证 | active/idle/degraded/offline/never_run |
| 备份系统 | ✅ 已验证 | G 盘每日 23:30 自动备份 |
| GitHub_Researcher 自动运行 | ❌ 未跑通 | 已注册，调度缺失 |
| Task Queue → Agent 执行 → 结果回写 | ❌ 未跑通 | 入队可以，消费链断 |
| Dashboard 接真实数据 | ❌ 未跑通 | v3.4/v4.0 均为随机数 |
| Events 回写链 | ❌ 未跑通 | 9 天无新事件 |
| Learning Loop 闭环 | ❌ 未跑通 | 有记录，无规则提炼 |

**P0 完成标准：** 至少有 3 条端到端链路能稳定复现。

---

## P1 — 闭环稳定

在 P0 跑通后推进：

- [ ] Scheduler 独立调度（不依赖心跳触发）
- [ ] Learning Loop 从失败中自动提炼规则
- [ ] alerts 路径统一验证（观察期 Day 3 验收）
- [ ] Dashboard 真数据接线（至少 Agent 状态 + 任务队列）

---

## P2 — 可选增强

P0/P1 稳定后再考虑：

- [ ] Docker 部署
- [ ] 代码清理（统一风格、类型注解）
- [ ] 单元测试覆盖
- [ ] 安全加固
- [ ] 社区建设 / 开源准备

---

## 已知技术债

1. **运行债** — 很多模块"注册存在"但没有进入真实运行（GitHub_Researcher、多个 Learning Agent）
2. **学习债** — 失败被记录但没转成可复用规则
3. **观测债** — Dashboard 造数、events 静默、路径分裂
4. **代码债** — 大量实验脚本混在主目录，_deprecated 清理不彻底
