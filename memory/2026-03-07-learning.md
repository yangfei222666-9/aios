## 今日 Top 5 项目

### 1. MetaGPT — ⭐ 64.8k
- **仓库：** https://github.com/FoundationAgents/MetaGPT
- **核心亮点：**
  - "软件公司即多智能体系统"——将 PM、架构师、工程师等角色分配给不同 LLM Agent，协作完成复杂任务
  - 提出 `Code = SOP(Team)` 哲学，将标准作业流程（SOP）物化为 Agent 协作协议
  - 2025年2月推出 MGX（MetaGPT X）自然语言编程产品，登顶 ProductHunt 周榜
  - 论文 AFlow（自动化 Agentic 工作流生成）在 ICLR 2025 获 oral（top 1.8%）
  - 支持 SPO（系统提示优化）和 AOT（思维链优化）

### 2. Letta (前 MemGPT) — ⭐ 21.4k
- **仓库：** https://github.com/letta-ai/letta
- **核心亮点：**
  - 专注"有状态 Agent"——AI 拥有持久记忆，能随时间学习和自我改进
  - 内置 Skills 系统 + Subagents 支持，预置高级记忆和持续学习技能包
  - 提供完整 Agent API（Python + TypeScript SDK），支持 memory_blocks 结构化记忆
  - 模型无关，推荐 Opus 4.5 / GPT-5.2，有公开模型排行榜
  - 定位：构建"民主化自我改进超级智能"的基础设施

### 3. OpenAI Agents SDK — ⭐ 19.4k
- **仓库：** https://github.com/openai/openai-agents-python
- **核心亮点：**
  - 轻量但强大的多 Agent 工作流框架，支持 100+ LLM（provider-agnostic）
  - 核心概念：Agents as Tools（Agent 可作为工具被其他 Agent 调用）、Handoffs（任务移交）
  - 内置 Guardrails（输入/输出安全校验）、Human-in-the-loop、Session 管理
  - 内置 Tracing（运行追踪、调试、优化）
  - 支持 Realtime Agents（语音 Agent）
  - 更新极活跃（58分钟前有提交）

### 4. MemOS — ⭐ 6.2k
- **仓库：** https://github.com/MemTensor/MemOS
- **核心亮点：**
  - 定位为 LLM 和 Agent 系统的"记忆操作系统"（Memory OS）
  - 统一管理长期记忆的 store / retrieve / manage 三大操作
  - 支持 KB（知识库）、多模态、工具记忆，企业级优化
  - 核心创新：**持久化 Skill 记忆**，实现跨任务技能复用与演化（Skill Memory for cross-task skill reuse and evolution）
  - 有学术论文支撑（arXiv 2507.03724）
  - 近期更新活跃（22小时前）

### 5. AIOS: AI Agent Operating System — ⭐ 5.3k
- **仓库：** https://github.com/agiresearch/AIOS
- **核心亮点：**
  - 将 LLM 嵌入操作系统内核，解决 Agent 调度、上下文切换、内存/存储/工具管理等 OS 级问题
  - 架构分层：AIOS Kernel（资源管理）+ AIOS SDK（Cerebrum，Agent 开发）
  - 支持 Computer-use Agent：VM Controller + MCP Server 沙箱环境
  - 四种部署模式：本地/远程/个人远程/虚拟化内核
  - A-MEM（Agentic Memory）：动态记忆网络，支持记忆关联和演化
  - 论文被 COLM 2025 接收

| 我们的设计 | 对标项目 | 优化方向 |
|---|---|---|
| Self-Improving Loop | Letta 持续学习 | 参考 Letta 的 memory_blocks 结构，将经验分类为 persona/human/task 三类，提升检索精度 |
| LanceDB 经验库 | MemOS Memory OS | MemOS 的 store/retrieve/manage 三层抽象值得借鉴，可为 LanceDB 加一层统一接口 |
| Evolution Score | MetaGPT AFlow | AFlow 自动化工作流生成可启发 Evolution Score 的自动调参——让 Agent 自己优化评分权重 |
| Bull vs Bear 对抗验证 | OpenAI Guardrails | OpenAI SDK 的 Guardrails 机制（输入/输出双向校验）可强化我们的对抗验证，加入结构化安全层 |

### 我们没有但值得借鉴的设计

1. **Tracing / 可观测性（OpenAI Agents SDK）**
   - 我们缺乏完整的 Agent 运行追踪系统
   - 建议：为每次决策循环记录 trace_id、输入、输出、工具调用链，存入 LanceDB，支持回溯调试

2. **SOP 物化（MetaGPT）**
   - MetaGPT 将工作流程编码为可复用的 SOP 对象
   - 建议：将我们的 64 卦决策逻辑封装为标准化 SOP 模板，支持不同场景切换不同卦象策略集

3. **Skill Memory 跨任务复用（MemOS）**
   - MemOS 的核心创新：技能不只是代码，而是带有记忆的可演化实体
   - 建议：在 LanceDB 经验库中为每个 Skill 增加"使用历史"和"成功率"字段，实现技能级别的 Evolution Score

4. **Agents as Tools / Handoffs（OpenAI SDK）**
   - Agent 可以把另一个 Agent 当工具调用，实现动态任务分发
   - 建议：我们的 Bull/Bear 对抗可升级为真正的双 Agent 架构，各自独立运行，通过 Handoff 协议交换结论

5. **Human-in-the-loop 标准化（OpenAI SDK）**
   - 内置人工介入机制，支持在 Agent 运行中途暂停等待人工确认
   - 建议：为高风险决策（如 Evolution Score 触发系统级改动）加入标准化的人工确认节点

6. **Computer-use 沙箱（AIOS agiresearch）**
   - VM Controller + MCP Server 实现安全的计算机操作
   - 建议：我们的 AIOS 如需扩展到自动化操作电脑，可参考此沙箱架构

## 新兴趋势总结

### 趋势一：记忆操作系统化（Memory OS）
- MemOS、Letta 都在将"记忆"从附属功能升级为独立的操作系统层
- 记忆不再是简单的向量检索，而是有生命周期、有演化能力的实体
- **我们的机会：** LanceDB + Evolution Score 已有基础，可向 Memory OS 方向演进

### 趋势二：Agent 可观测性（Observability）
- OpenAI SDK 内置 Tracing，RagaAI Catalyst（16.1k⭐）专注 Agent 监控评估
- 生产级 Agent 必须可追踪、可调试、可评估
- **我们的机会：** 补充 Tracing 层，让 Self-Improving Loop 的每次迭代都有完整记录

### 趋势三：工作流自动生成（Agentic Workflow Generation）
- MetaGPT AFlow 在 ICLR 2025 获 oral，自动生成最优 Agent 工作流
- Agent 不只执行任务，还能优化自己的执行流程
- **我们的机会：** Evolution Score 可扩展为工作流级别的自动优化

### 趋势四：多 Agent 协作标准化
- OpenAI SDK 的 Handoffs、MetaGPT 的 SOP、Letta 的 Subagents 都在标准化多 Agent 协作协议
- 单一 Agent 架构正在被多 Agent 协作网络取代
- **我们的机会：** Bull vs Bear 是天然的双 Agent 架构，可扩展为更大的 Agent 网络

### 趋势五：Skill 系统独立化
- MemOS 的 Skill Memory、Letta 的 Skills、OpenClaw 生态的 Skill 系统都在将技能作为一等公民
- 技能不只是函数，而是有记忆、有版本、可演化的知识单元
- **我们的机会：** 为每个 Skill 建立独立的 Evolution Score，实现技能粒度的自我改进
