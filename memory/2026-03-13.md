# 2026-03-13 学习日志

## GitHub 研究日报

### 今日发现（5个项目）

1. **agiresearch/AIOS** ⭐5.3k — https://github.com/agiresearch/AIOS
   学术级 Agent OS，内核层抽象 LLM 调度 + Context 切换 + Memory 管理，配套 A-MEM 用 Zettelkatten 原则做动态记忆演化

2. **letta-ai/letta**（原 MemGPT）— https://github.com/letta-ai/letta
   有状态 Agent 平台，分层记忆（memory_blocks），Agent 可自主读写记忆并持续自我改进

3. **aden-hive/hive**（YC 孵化）— https://github.com/aden-hive/hive
   结果驱动自进化：对话定义目标 → 自动生成节点图 → 失败时捕获数据自动修复重部署

4. **ag2ai/ag2**（原 AutoGen）— https://github.com/ag2ai/ag2
   多 Agent 协作标杆：Swarm、GroupChat、NestedChat、SequentialChat 等编排模式

5. **microsoft/agent-framework** — https://github.com/microsoft/agent-framework
   Graph-based Workflow + OpenTelemetry 可观测性 + 检查点恢复

---

### 与太极OS 的差距

1. 记忆系统缺少自动演化（无关联发现、无冷记忆衰减）
2. 缺 Graph-based 工作流（线性 Task Queue 无法表达分支/并行）
3. 失败自愈偏弱（事后总结 vs 实时捕获+自动修复）
4. 多 Agent 协作模式单一
5. 可观测性缺标准化 tracing/metrics

---

### ⭐ 最值得借鉴的 2 个设计（重点）

**1. 记忆演化与自主关联**（AIOS A-MEM + Letta）
- 记忆不只是存取，而是会"生长"——自动建立关联、合并冗余、衰减冷记忆
- 改进步骤：
  1. Memory Server `/ingest` 增加"关联发现"，新记忆写入时自动与 top-3 相似记忆建立双向链接，存入 `links` 字段
  2. Self-Improving Loop 加入记忆整合阶段，Agent 可主动合并/升级/淘汰旧记忆
  3. 记忆条目增加 `access_count` 和 `last_evolved` 时间戳，支持活跃度衰减

**2. 失败自愈闭环**（Hive + Microsoft Agent Framework）
- 失败→诊断→修复→恢复 全自动闭环，不是事后总结而是实时自愈
- 改进步骤：
  1. Task Queue 执行层增加检查点快照，失败时从最近检查点恢复
  2. 失败事件自动打包上下文（错误日志 + 输入参数 + 检查点），写入 `incidents/` 目录，触发 lesson-extractor
  3. 对高频失败模式，自动生成修复配置补丁草稿，人工确认后应用
