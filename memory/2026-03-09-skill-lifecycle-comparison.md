# hermes-agent ↔️ 太极OS Skill 生命周期对照表

**研究时间：** 2026-03-09  
**研究目的：** 理解 hermes-agent 的 Skill 自动创建机制，为太极OS 设计 Skill 自动创建 MVP 提供基础

---

## 1. Skill 发现（Discovery）

### hermes-agent 现在怎么做
- **触发方式：** 任务完成后自动分析
- **发现机制：** 
  - 从成功的任务执行轨迹中识别可复用模式
  - 通过 LLM 分析任务步骤，提取通用流程
  - 检测重复出现的操作序列（如"读文件 → 分析 → 写报告"）
- **判断标准：**
  - 任务成功完成
  - 操作步骤 ≥ 3 个
  - 具有明确的输入输出
  - 可能在未来重复使用

### 太极OS 现在怎么做
- **触发方式：** 手动识别
- **发现机制：**
  - 人工观察重复任务
  - 手动决定是否需要新 Skill
  - 没有自动模式识别
- **判断标准：**
  - 完全依赖人工判断
  - 没有量化标准

### 差距是什么
1. **自动化程度：** hermes-agent 自动发现，太极OS 手动识别
2. **模式识别：** hermes-agent 有 LLM 驱动的模式提取，太极OS 没有
3. **触发时机：** hermes-agent 任务完成即分析，太极OS 事后回顾
4. **量化标准：** hermes-agent 有明确判断标准，太极OS 没有

---

## 2. Skill 生成（Generation）

### hermes-agent 现在怎么做
- **生成者：** LLM（通过 prompt 模板）
- **生成流程：**
  1. 提取任务执行轨迹
  2. 识别关键步骤和参数
  3. 生成 Skill 代码（Python 函数）
  4. 生成 Skill 文档（描述、参数、返回值）
  5. 生成使用示例
- **输出格式：**
  - Python 函数（带类型注解）
  - Docstring（描述 + 参数 + 返回值）
  - 使用示例（代码片段）
- **模板化：** 使用标准化的 Skill 模板，确保一致性

### 太极OS 现在怎么做
- **生成者：** 人工编写
- **生成流程：**
  1. 手动写 Python 脚本
  2. 手动写 SKILL.md 文档
  3. 手动测试
  4. 手动集成到系统
- **输出格式：**
  - Python 脚本（无统一模板）
  - SKILL.md（格式不完全统一）
  - 使用示例（可能缺失）
- **模板化：** 有 SKILL.md 模板，但代码部分无模板

### 差距是什么
1. **生成速度：** hermes-agent 秒级生成，太极OS 需要小时级
2. **一致性：** hermes-agent 模板化保证一致性，太极OS 手写容易不一致
3. **文档同步：** hermes-agent 代码和文档同时生成，太极OS 容易脱节
4. **示例质量：** hermes-agent 自动生成真实示例，太极OS 手写示例可能过时

---

## 3. Skill 验证（Validation）

### hermes-agent 现在怎么做
- **验证层级：**
  1. **语法验证：** Python AST 解析，确保代码可执行
  2. **安全验证：** 检查危险操作（如 `os.system`、`eval`）
  3. **功能验证：** 在沙箱中执行测试用例
  4. **效果验证：** 对比原始任务和 Skill 执行结果
- **沙箱隔离：** 使用 Docker/Firecracker 隔离执行
- **失败处理：**
  - 语法错误 → 重新生成
  - 安全问题 → 拒绝并记录
  - 功能失败 → 标记为 draft，等待改进
  - 效果不佳 → 降低优先级

### 太极OS 现在怎么做
- **验证层级：**
  1. **手动测试：** 人工运行脚本
  2. **观察结果：** 人工判断是否符合预期
  3. **无自动化验证**
- **沙箱隔离：** 无，直接在宿主机执行
- **失败处理：**
  - 手动修复
  - 无自动重试
  - 无失败记录机制

### 差距是什么
1. **自动化：** hermes-agent 全自动验证，太极OS 手动测试
2. **安全性：** hermes-agent 有安全检查 + 沙箱，太极OS 无
3. **可靠性：** hermes-agent 多层验证，太极OS 单层人工判断
4. **失败恢复：** hermes-agent 自动重试/降级，太极OS 手动修复

---

## 4. Skill 注册/索引（Registration & Indexing）

### hermes-agent 现在怎么做
- **注册方式：**
  - 验证通过后自动注册到 Skills Hub
  - 生成唯一 Skill ID
  - 记录创建时间、版本、来源任务
- **索引机制：**
  - **语义索引：** 使用 embedding 向量化 Skill 描述
  - **标签索引：** 自动提取关键词标签
  - **分类索引：** 按功能分类（文件操作、数据分析、API 调用等）
- **检索方式：**
  - 自然语言查询（"帮我分析 CSV 文件"）
  - 标签过滤
  - 相似度排序
- **版本管理：**
  - 每次改进生成新版本
  - 保留历史版本
  - 可回滚到旧版本

### 太极OS 现在怎么做
- **注册方式：**
  - 手动创建 Skill 文件夹
  - 手动写 SKILL.md
  - 手动放到 skills/ 目录
- **索引机制：**
  - 文件系统目录结构
  - SKILL.md 的 description 字段
  - 无语义索引
- **检索方式：**
  - 手动查找文件夹
  - 搜索 SKILL.md 文本
  - 无智能推荐
- **版本管理：**
  - 无版本控制
  - 修改直接覆盖
  - 无历史记录

### 差距是什么
1. **智能化：** hermes-agent 语义索引 + 智能推荐，太极OS 文件系统查找
2. **自动化：** hermes-agent 自动注册，太极OS 手动创建
3. **版本管理：** hermes-agent 完整版本历史，太极OS 无版本控制
4. **可发现性：** hermes-agent 自然语言查询，太极OS 需要知道 Skill 名字

## 5. 使用后自改进（Self-Improvement）

### hermes-agent 现在怎么做
- **改进触发：**
  - 每次 Skill 使用后自动分析
  - 收集执行时间、成功率、错误信息
  - 用户反馈（显式或隐式）
- **改进机制：**
  1. **性能优化：** 分析慢步骤，生成优化版本
  2. **错误修复：** 从失败案例中学习，修复 bug
  3. **参数调优：** 根据使用模式调整默认参数
  4. **功能扩展：** 识别常见变体，扩展 Skill 能力
- **改进验证：**
  - A/B 测试（旧版本 vs 新版本）
  - 成功率对比
  - 性能对比
  - 自动选择更好的版本
- **持久化：**
  - 周期性 nudge 机制
  - 将改进写回 Skills Hub
  - 更新 embedding 索引

### 太极OS 现在怎么做
- **改进触发：**
  - 手动发现问题
  - 手动决定是否改进
- **改进机制：**
  - 手动修改代码
  - 手动测试
  - 手动部署
- **改进验证：**
  - 手动对比
  - 无 A/B 测试
  - 无量化指标
- **持久化：**
  - 直接覆盖旧版本
  - 无改进历史记录

### 差距是什么
1. **自动化：** hermes-agent 自动改进，太极OS 手动修改
2. **数据驱动：** hermes-agent 基于使用数据优化，太极OS 凭经验修改
3. **验证机制：** hermes-agent A/B 测试，太极OS 无对比验证
4. **持续性：** hermes-agent 持续改进，太极OS 一次性修改

---

## 总结：核心差距

### 最大差距（P0）
**Skill 自动创建机制缺失**
- hermes-agent：任务完成 → 自动发现 → 自动生成 → 自动验证 → 自动注册
- 太极OS：手动识别 → 手动编写 → 手动测试 → 手动部署

### 次要差距（P1）
1. **语义索引缺失** — 无法通过自然语言查询 Skill
2. **版本管理缺失** — 无法回滚、对比、追踪改进历史
3. **自我改进缺失** — Skill 不会从使用中学习和优化

### 独特优势（保持）
1. **Reality Ledger** — hermes-agent 没有操作审计链
2. **Evolution Score** — hermes-agent 没有系统进化评分
3. **易经状态引擎** — 独特的状态判断框架
4. **Task Queue + 心跳调度** — hermes-agent 用 cron
