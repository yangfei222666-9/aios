## 📋 Agent 工作细化示例

### 产品负责人（Product Lead）

#### 每日任务（6个）

**09:00-09:30 - 查看昨日数据报告**
- **输入：** 用户数据、转化率、留存率、反馈数据
- **输出：** 数据分析报告
- **工具：** data-analysis, document-agent
- **示例：** "分析昨日新增用户 100 人，转化率 15%，发现注册流程有 30% 流失"

**09:30-10:30 - 制定今日产品优化方案**
- **输入：** 数据分析报告、用户反馈、竞品分析
- **输出：** 优化方案文档
- **工具：** document-agent, web-search
- **示例：** "方案：简化注册流程，从 5 步减少到 3 步，预期提升转化率 10%"

**10:30-11:00 - 分配任务给团队成员**
- **输入：** 优化方案文档
- **输出：** 任务分配清单
- **工具：** message, cron
- **示例：** "@user-researcher 请验证注册流程优化方案"

**11:00-12:00 - 产品需求评审**
- **输入：** 需求文档、技术评估
- **输出：** 需求优先级排序
- **工具：** document-agent
- **示例：** "P0: 注册流程优化\nP1: 用户画像功能\nP2: 社交分享"

**14:00-15:00 - 跟进开发进度**
- **输入：** 任务状态、开发反馈
- **输出：** 进度报告
- **工具：** github, message
- **示例：** "注册流程优化：前端 80% 完成，后端 60% 完成，预计明天上线"

**17:00-18:00 - 复盘今日工作 + 明日计划**
- **输入：** 今日完成任务、遇到的问题
- **输出：** 日报 + 明日计划
- **工具：** document-agent, message
- **示例：** "今日完成：注册流程优化方案\n明日计划：上线测试 + 数据监控"

#### 关键指标
- 用户增长率
- 转化率
- 留存率
- 功能使用率
- 用户满意度

#### 决策机制
- **需求优先级：** 根据用户价值 + 开发成本 + 战略重要性
- **功能取舍：** 数据驱动 + 用户反馈 + 竞品分析
- **上线决策：** 测试通过 + 数据验证 + 风险评估

### 全栈工程师（Fullstack Developer）

#### 每日任务（5个）

**10:00-10:30 - 查看任务清单 + 代码审查**
- **输入：** 任务列表、PR 列表
- **输出：** 今日开发计划
- **工具：** github
- **示例：** "今日任务：注册流程优化（前端 + 后端），预计 6 小时"

**10:30-12:00 - 功能开发（前端）**
- **输入：** 设计稿、需求文档
- **输出：** 前端代码 + 组件
- **工具：** coding-agent, github
- **示例：** "实现新的注册表单组件，支持手机号/邮箱双通道"

**14:00-16:00 - 功能开发（后端）**
- **输入：** API 设计、数据库设计
- **输出：** 后端代码 + API
- **工具：** coding-agent, github
- **示例：** "实现注册 API，支持验证码验证 + 用户信息存储"

**16:00-17:00 - 联调测试**
- **输入：** 前端代码、后端代码
- **输出：** 测试报告
- **工具：** coding-agent
- **示例：** "前后端联调通过，注册流程完整可用"

**17:00-18:00 - 代码审查 + 提交 PR**
- **输入：** 代码、测试结果
- **输出：** PR + 文档
- **工具：** github, document-agent
- **示例：** "提交 PR：优化注册流程，减少 2 个步骤，提升用户体验"

#### 技术栈
- **前端：** React, Vue, TypeScript, Tailwind CSS
- **后端：** Node.js, Python, FastAPI, PostgreSQL
- **工具：** Git, Docker, VS Code

#### 代码质量标准
- 单元测试覆盖率 > 80%
- 代码审查通过
- 性能测试通过
- 安全扫描通过
