## 🔄 工作流程

### 产品开发流程

从需求到上线的完整流程

1. **ceo** - 派发任务给产品负责人 (08:00)
2. **product-lead** - 分析需求，制定方案 (09:00)
3. **user-researcher** - 用户研究，验证需求 (09:30)
4. **ux-designer** - 设计交互原型 (10:00)
5. **fullstack-dev** - 功能开发实现 (11:00)
6. **qa-automation** - 自动化测试 (14:00)
7. **devops-engineer** - 部署上线 (15:00)
8. **technical-writer** - 更新文档 (16:00)
9. **content-strategist** - 发布更新公告 (17:00)

### 增长实验流程

数据驱动的增长实验

1. **growth-lead** - 提出增长假设 (09:00)
2. **data-analyst** - 数据分析，验证假设 (10:00)
3. **product-lead** - 设计实验方案 (11:00)
4. **fullstack-dev** - 实现实验功能 (13:00)
5. **acquisition-specialist** - 投放测试流量 (15:00)
6. **data-analyst** - 分析实验结果 (17:00)
7. **growth-lead** - 决策是否推广 (18:00)

## 💬 沟通规则

### 调用格式

通过前缀调用Agent

**示例:** @product-lead 请分析用户反馈，制定优化方案

### 任务传递

Agent完成任务后自动传递给下一个Agent

**示例:** product-lead完成需求分析 → 自动通知fullstack-dev开始开发

### 汇报机制

每个Agent完成任务后向上级汇报

**示例:** fullstack-dev完成开发 → 汇报给product-lead → 汇报给ceo

### 协作模式

Agent之间可以互相请求协助

**示例:** fullstack-dev遇到技术问题 → 请求backend-specialist协助

## 🚀 使用方法

### 1. 部署团队

```bash
python deploy_ai_team.py
```

### 2. 调用 Agent

```python
# 通过 Agent ID 调用
@product-lead 请分析用户反馈，制定优化方案

# 通过角色名调用
@产品负责人 请分析用户反馈，制定优化方案
```

### 3. 任务流转

Agent 完成任务后会自动传递给下一个 Agent：

```
CEO → 产品负责人 → 全栈工程师 → QA自动化 → DevOps工程师
```

### 4. 查看状态

```bash
# 查看所有 Agent 状态
python check_team_status.py

# 查看单个 Agent 状态
python check_agent_status.py product-lead
```

## 📝 注意事项

1. **Agent 专业化** - 每个 Agent 有明确的职责和专长
2. **协作流程** - Agent 之间通过任务传递协作
3. **汇报机制** - 每个 Agent 完成任务后向上级汇报
4. **工作时间** - 每个 Agent 有固定的工作时间
