## 集成前必须通过的闸门

### 1. 安全审查（必须全部通过）

- [ ] **Dry-Run 模式** - 所有操作先模拟，不实际执行
- [ ] **审计日志** - 每个操作记录到 `rpa_audit.jsonl`
- [ ] **人工确认点** - 高风险操作（点击、输入）需人工确认
- [ ] **操作白名单** - 只允许预定义的安全操作
- [ ] **区域限制** - 禁止操作系统关键区域（任务栏、系统托盘）
- [ ] **窗口限制** - 只允许操作指定应用窗口
- [ ] **回滚机制** - 操作失败时自动回滚

### 2. 风险评估（必须全部低风险）

- [ ] **误操作风险** - 点错、输错、操作错对象的概率 < 1%
- [ ] **数据泄露风险** - 不会截取敏感信息（密码、私钥）
- [ ] **系统破坏风险** - 不会删除、修改系统文件
- [ ] **权限滥用风险** - 不会提权、不会绕过安全机制

### 3. 功能验证（必须全部通过）

- [ ] **OCR 准确率** - 中文识别率 > 95%，英文识别率 > 98%
- [ ] **文本定位准确率** - 定位误差 < 5 像素
- [ ] **工作流成功率** - 连续 10 次执行成功率 > 90%
- [ ] **异常处理** - 所有异常都能被捕获和记录

### 4. 集成测试（必须全部通过）

- [ ] **隔离测试** - 在独立环境中测试 7 天
- [ ] **压力测试** - 连续执行 100 次工作流无崩溃
- [ ] **兼容性测试** - 在不同分辨率、不同窗口大小下测试
- [ ] **回归测试** - 不影响现有 AIOS 功能

### 5. 文档完善（必须全部完成）

- [ ] **安全使用指南** - 明确禁止的操作
- [ ] **故障排查手册** - 常见问题和解决方案
- [ ] **审计日志格式** - 标准化日志格式
- [ ] **应急响应流程** - 误操作后的补救措施

## 集成后的运行规则

### 1. 默认禁用

- RPA 功能默认禁用
- 需要手动启用：`rpa.enable(confirm=True)`

### 2. 人工确认

- 所有高风险操作需人工确认
- 确认超时（30 秒）自动取消

### 3. 操作限制

- 禁止操作系统关键区域
- 禁止操作未授权的应用窗口
- 禁止执行未审查的工作流

### 4. 审计日志

- 所有操作记录到 `rpa_audit.jsonl`
- 日志保留 30 天
- 定期审查异常操作

### 5. 紧急停止

- 鼠标移到屏幕左上角立即停止
- 快捷键 `Ctrl+Alt+Q` 紧急停止
- 所有操作可回滚
