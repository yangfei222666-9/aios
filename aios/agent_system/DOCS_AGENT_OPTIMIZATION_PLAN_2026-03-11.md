## P2：优化 Docs_Demo_Writer

### 问题

**59.75s 太贴边** - 接近下一次随机抖动就超时

### 目标

**降到 45s 左右**，留出安全余量

### 优化方式

#### 方案 A：缩小 demo 范围（推荐）

**当前：** 3 个场景（文件监控、API 健康检查、日志分析）

**优化：** 每次只写 1 个场景

**实施：**
```python
# 修改 learning_agents.py
{
    "name": "Docs_Demo_Writer",
    "tasks": [
        "本次只处理：docs/examples/ 目录下的 demo 文档",
        "只补内容：选择以下一个场景：文件监控 OR API 健康检查 OR 日志分析",
        "输出文件：docs/examples/<scenario>.md（单个文件）",
        "不允许：一次写多个场景，不允许修改其他文档",
        "完成标准：单个 demo 包含场景描述、代码示例、运行步骤、预期输出"
    ]
}
```

#### 方案 B：限制示例数量

**当前：** 每个场景包含多个示例（基础 + 高级）

**优化：** 每个场景只包含 1 个基础示例

**实施：**
```python
"只补内容：每个场景只写 1 个基础示例，不要写高级示例"
```

#### 方案 C：固定输出模板

**当前：** 自由发挥，可能补充背景说明

**优化：** 固定模板，不要顺手补充

**实施：**
```python
"输出模板：
1. 场景描述（不超过 3 句话）
2. 代码示例（不超过 50 行）
3. 运行步骤（不超过 3 步）
4. 预期输出（不超过 5 行）
不要补充背景说明、最佳实践、扩展阅读"
```

### 推荐方案

**方案 A（缩小范围）** - 最直接有效

### 验收标准

1. 耗时降到 45s 以下
2. 输出质量不降低
3. 稳定通过 60s 阈值

## P3：修 Docs_FAQ_Writer prompt

### 问题

**输出交互式提示** - "请提供信息"而非完整 FAQ

### 原因

Prompt 约束不够硬，coder 认为需要用户输入

### 目标

**去交互化** - 直接生成完整 FAQ

### 优化方式

**修改 learning_agents.py：**

```python
{
    "name": "Docs_FAQ_Writer",
    "tasks": [
        "本次只处理：docs/FAQ.md",
        "只补内容：至少 10 个常见问题和解答",
        "输出文件：docs/FAQ.md",
        "不允许：等待用户输入、提出反问、要求用户确认",
        "完成标准：直接生成包含 10 个问题的完整 FAQ，每个问题有清晰解答和示例",
        "执行方式：不要交互，直接按给定范围生成最终内容"
    ]
}
```

### 关键约束

- **不要提出反问**
- **不要等待用户确认**
- **直接按给定范围生成最终内容**

### 验收标准

1. 输出完整 FAQ（不是交互式提示）
2. 包含至少 10 个问题
3. 每个问题有清晰解答
